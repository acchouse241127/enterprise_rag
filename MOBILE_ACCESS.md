# 手机访问本地项目指南

## 局域网 IP 地址

你的电脑局域网 IP 可能是以下之一（请确认）：
- **192.168.50.205** (主要)
- 192.168.50.1
- 172.30.80.1

### 如何获取正确的 IP

在电脑上运行命令查看：
```bash
# Windows (PowerShell)
Get-NetIPAddressConfiguration | Select-Object InterfaceAlias, IPv4Address

# 或查看路由器管理页面查看连接设备列表
```

## 访问地址

确保手机和电脑在**同一个 WiFi 网络**下，然后访问：

### SPA (Docker 部署)
```
http://192.168.50.205:3000
```

### Vite Dev (开发模式)
```
http://192.168.50.205:3003
```

### 后端 API
```
http://192.168.50.205:8000
```

## 如果无法访问

### 1. 检查防火墙

**Windows 防火墙设置：**
1. 打开"Windows 安全中心"
2. 点击"防火墙和网络保护"
3. 选择"高级设置"
4. 添加入站规则：
   - 端口：3000, 3003, 8000
   - 协议：TCP
   - 操作：允许连接

### 2. 验证端口监听

检查 Docker 容器是否正确绑定：
```bash
docker ps --format "table {{.Names}}\t{{.Ports}}"
```

应该看到：
```
enterprise_rag_spa        0.0.0.0:3000->80/tcp
enterprise_rag_backend    0.0.0.0:8000->8000/tcp
```

`0.0.0.0` 表示监听所有网络接口。

### 3. 测试局域网访问

在电脑上用局域网 IP 访问：
```bash
curl http://192.168.50.205:3000
```

如果成功，说明网络配置正确，问题在手机端。

### 4. 手机端检查

- 确认手机和电脑在**同一个 WiFi**
- 清除手机浏览器缓存
- 尝试使用其他浏览器
- 检查手机是否启用了 WiFi 助手或广告拦截

## 快速测试

1. 在电脑浏览器访问：`http://localhost:3000` - 确认本地可访问
2. 在电脑浏览器访问：`http://192.168.50.205:3000` - 确认局域网可访问
3. 在手机浏览器访问：`http://192.168.50.205:3000` - 测试手机访问

如果步骤 1 成功但步骤 2 失败，说明是防火墙问题。
如果步骤 2 成功但步骤 3 失败，检查手机是否在同一 WiFi。
