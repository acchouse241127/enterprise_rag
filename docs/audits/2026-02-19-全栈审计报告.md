### 2.2 详细发现

#### 🟠 重要：分享链接格式与 Streamlit 路由不匹配

**位置**：`frontend/pages/5_对话管理.py` 第 269-271 行

```python
st.text_input(
    "分享链接",
    value=f"/share/{conv['share_token']}",
    ...
)
```

**问题**：展示的链接为 `/share/{token}`，而 Streamlit 分享页 `10_分享查看.py` 依赖查询参数 `?token=xxx`。用户复制该相对路径访问会 404，无法查看分享内容。

**建议动作**：
- 将分享链接改为完整 URL：`{BACKEND_URL 或当前域}/分享查看?token={token}`
- 或增加「复制分享链接」按钮，生成可访问的完整 URL（如 `http://localhost:8501/分享查看?token=xxx`）

---

#### 🟡 建议：分享查看页需在侧边栏可见

**位置**：Streamlit 多页结构

**问题**：`10_分享查看.py` 作为独立页，通常通过侧边栏进入；他人通过分享链接访问时，需直接打开带 `?token=xxx` 的 URL。需在文档或引导中说明「分享链接使用方法」。

**建议动作**：在用户手册或对话管理页增加简短说明：「复制完整链接（含 ?token=）在新标签页打开即可查看」。

- URL 文档的 reparse 存在已知 Bug

### 3.2 详细发现

#### 🔴 阻塞（已知）：URL 文档 reparse 返回「文件不存在」

**位置**：`backend/app/services/document_service.py` 第 261-264 行

```python
path = Path(doc.file_path)
if not path.exists():
    return None, "文件不存在"
```

**问题**：URL 文档的 `file_path` 存的是 URL 字符串（如 `https://example.com/page`），`Path(url).exists()` 恒为 False，导致 reparse 始终失败。

**建议动作**：在 `reparse` 开头增加 `file_type == "url"` 分支，对 URL 文档入队 `parse_and_index.delay(document_id)` 或调用 `parse_and_index_sync` 中已有的 URL 处理逻辑，而非走本地文件路径判断。

---

#### 🟡 建议：CORS 默认仅 localhost:3000

**位置**：`backend/app/config.py` 第 26 行

```python
cors_origins: str = "http://localhost:3000"
```

**问题**：Streamlit 默认 8501，独立 SPA 开发时 3000。若未配置 `CORS_ORIGINS`，8501 的 Streamlit 可能被 CORS 拒绝（若通过不同端口代理）。

**建议动作**：默认值增加 `http://localhost:8501`，或文档明确说明需在 `.env` 中配置 `CORS_ORIGINS`。
