## 二、快速启动

### 本地开发模式

```powershell
# 1. 进入项目目录
cd enterprise_rag

# 2. 启动数据库
docker compose up -d postgres chromadb

# 3. 等待约 10 秒后，启动后端
cd backend
pip install -r requirements.txt
python scripts/init_db.py   # 首次需初始化
uvicorn main:app --reload --host 0.0.0.0 --port 8000

# 4. 新开终端，启动前端
cd frontend
pip install -r requirements.txt
streamlit run 系统介绍.py --server.port 8501
```

### 全栈 Docker 模式（含 Redis、Worker）

```powershell
cd enterprise_rag
docker compose --profile full up -d
# 访问：前端 http://localhost:8501，后端 http://localhost:8000
```

## 四、使用注意事项

1. **分享链接**：对话管理页的分享链接现已为完整 URL（如 `http://localhost:8501/分享查看?token=xxx`），复制后在新标签页打开即可查看，无需登录。生产环境需在 `.env` 中配置 `FRONTEND_BASE_URL` 为实际前端域名。

2. **URL 文档重新解析**：从网址导入的文档，点击「重新解析」不再报「文件不存在」，会正常入队由 Worker 执行。

3. **环境变量**：生产部署前请按 README「生产部署必改项」修改 `JWT_SECRET_KEY`、`env`、`POSTGRES_PASSWORD`、`CORS_ORIGINS` 等。

4. **测试账号**：`scripts/init_db.py` 初始化后可选用 `admin` / `password123` 登录。
