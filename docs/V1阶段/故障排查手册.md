# Enterprise RAG 故障排查手册

## 快速诊断

### 1. 检查服务状态

```bash
# 检查所有容器状态
docker-compose -f docker-compose.yml --profile full ps

# 检查后端健康
curl http://localhost:8000/api/system/health

# 检查 ChromaDB
curl http://localhost:8001/api/v1/heartbeat

# 检查 Redis
redis-cli ping
```

## 常见问题

### 问题 1：文档上传后一直显示"等待中"

**症状**：文档状态卡在 `pending`，不进入 `parsing`

**可能原因**：
1. Celery Worker 未启动
2. Redis 连接失败
3. Worker 进程崩溃

**排查步骤**：

```bash
# 1. 检查 Worker 是否运行
docker-compose -f docker-compose.yml --profile full logs worker

# 2. 检查 Redis 连接
docker exec -it enterprise_rag_redis redis-cli ping

# 3. 重启 Worker
docker-compose -f docker-compose.yml --profile full restart worker

# 4. 手动触发解析（调试用）
curl -X POST http://localhost:8000/api/documents/{doc_id}/reparse
```

---

### 问题 2：问答返回"未检索到足够知识"

**症状**：问答接口返回空答案

**可能原因**：
1. 知识库没有已向量化的文档
2. ChromaDB 连接失败
3. 检索阈值设置过严

**排查步骤**：

```bash
# 1. 检查文档状态
curl http://localhost:8000/api/knowledge-bases/{kb_id}/documents

# 2. 检查 ChromaDB 连接
curl http://localhost:8001/api/v1/collections

# 3. 检查 Collection 是否存在
curl "http://localhost:8001/api/v1/collections/kb_{kb_id}"

# 4. 调整检索阈值（在 .env 中）
DYNAMIC_THRESHOLD_FALLBACK=0.9  # 更宽松
```

---

### 问题 3：LLM 调用失败

**症状**：问答返回 `llm_failed: true`

**可能原因**：
1. API Key 无效
2. API 配额用尽
3. 网络连接问题
4. 模型服务不可用

**排查步骤**：

```bash
# 1. 检查 LLM 配置
curl http://localhost:8000/api/system/health

# 2. 测试 API 连接
curl https://api.deepseek.com/v1/models \
  -H "Authorization: Bearer YOUR_API_KEY"

# 3. 检查后端日志
docker-compose -f docker-compose.yml --profile full logs backend | grep -i llm

# 4. 降级测试（使用其他模型）
# 修改 .env 中的 LLM_PROVIDER 和相关配置
```

---

### 问题 4：前端无法连接后端

**症状**：SPA 显示"网络错误"或"未登录"

**可能原因**：
1. CORS 配置问题
2. 后端未启动
3. 代理配置错误

**排查步骤**：

```bash
# 1. 直接测试后端
curl http://localhost:8000/api/system/health

# 2. 检查 CORS 配置
# 在 .env 中确认
CORS_ORIGINS=http://localhost:3000,http://localhost:8501

# 3. 检查 Nginx 代理（如果在 Docker 中）
docker-compose -f docker-compose.yml --profile full logs spa

# 4. 检查浏览器控制台
# 打开 F12 -> Network 查看请求状态
```

---

### 问题 5：文件上传失败

**症状**：上传返回错误或文件丢失

**可能原因**：
1. 文件大小超限
2. 存储目录权限问题
3. 磁盘空间不足

**排查步骤**：

```bash
# 1. 检查文件大小限制
# .env 中配置
MAX_FILE_SIZE_MB=200

# 2. 检查上传目录
ls -la data/uploads/

# 3. 检查磁盘空间
df -h

# 4. 检查目录权限
chmod -R 755 data/uploads/
```

---

### 问题 6：向量检索慢

**症状**：问答响应时间超过 10 秒

**可能原因**：
1. Reranker 模型加载慢
2. 向量库数据量大
3. 服务器资源不足

**排查步骤**：

```bash
# 1. 检查 Reranker 状态
# 可临时禁用
RERANKER_ENABLED=false

# 2. 减少候选数量
RERANKER_CANDIDATE_K=8

# 3. 检查资源占用
docker stats

# 4. 使用低延迟策略
# 在问答请求中传入 strategy=low_latency
```

## 数据恢复

```bash
# 1. 停止服务
docker-compose -f docker-compose.yml --profile full down

# 2. 恢复数据
python scripts/restore.py ./backups/20260221_120000 --confirm

# 3. 重启服务
docker-compose -f docker-compose.yml --profile full up -d
```
