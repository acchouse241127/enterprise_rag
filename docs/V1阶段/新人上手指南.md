# Enterprise RAG 新人上手指南

欢迎加入 Enterprise RAG 项目！本指南将帮助你快速搭建开发环境并了解项目结构。

---

## 1. 环境准备

### 1.1 必需软件

| 软件 | 版本要求 | 用途 |
|------|----------|------|
| Docker Desktop | 20.0+ | 容器化部署 |
| Python | 3.11+ | 后端开发 |
| Node.js | 18.0+ | 前端开发 |
| Git | 最新版 | 版本控制 |

### 1.2 克隆项目

```bash
git clone [项目地址]
cd enterprise_rag
```

### 1.3 配置环境变量

```bash
# 复制示例配置
cp .env.example .env

# 编辑配置（必须修改以下项）
# - LLM_API_KEY: 你的 LLM API Key
# - JWT_SECRET_KEY: 随机字符串（生产环境）
```

## 2. 启动项目

### 2.1 使用 Docker（推荐）

```bash
# 启动所有服务
docker-compose -f docker-compose.yml --profile full up -d

# 查看服务状态
docker-compose -f docker-compose.yml --profile full ps

# 查看日志
docker-compose -f docker-compose.yml --profile full logs -f backend
```

### 2.2 本地开发模式

**后端**：

```bash
cd backend

# 创建虚拟环境
python -m venv venv
source venv/bin/activate  # Windows: venv\Scripts\activate

# 安装依赖
pip install -r requirements.txt

# 启动后端
uvicorn main:app --reload --port 8000

# 启动 Worker（新终端）
celery -A app.core.celery_app worker --loglevel=info
```

**前端**：

```bash
cd frontend_spa

# 安装依赖
npm install

# 启动开发服务器
npm run dev
```

### 2.3 访问地址

| 服务 | 地址 |
|------|------|
| SPA 前端 | http://localhost:3000 |
| Streamlit 前端 | http://localhost:8501 |
| API 文档 | http://localhost:8000/docs |
| ChromaDB | http://localhost:8001 |

## 4. 核心流程

### 4.1 文档处理流程

```
用户上传 → API 创建 Document(status=pending)
         → Celery 任务入队
         → Worker 消费
         → 解析(parse) → 分块(chunk) → 向量化(embed) → 入库(upsert)
         → 更新 status=vectorized
```

### 4.2 问答流程

```
用户提问 → 多路召回(向量+关键词) → 去重 → Reranker 重排
        → 构建 Prompt → LLM 生成 → 插入引用 → 返回答案
```

## 5. 开发指南

### 5.1 添加新 API 端点

1. 在 `backend/app/api/` 创建或编辑路由文件
2. 在 `backend/app/schemas/` 定义请求/响应 Schema
3. 在 `backend/app/services/` 实现业务逻辑
4. 在 `backend/app/api/__init__.py` 注册路由

### 5.2 添加新的文档解析器

1. 在 `backend/app/document_parser/` 创建解析器
2. 在 `backend/app/document_parser/__init__.py` 注册
3. 在 `backend/app/services/document_service.py` 使用

### 5.3 修改前端页面

1. 在 `frontend_spa/src/pages/` 编辑页面组件
2. 如需新 API，在 `frontend_spa/src/api.ts` 添加
3. 在 `frontend_spa/src/App.tsx` 添加路由
