#### ECC 规则：`python-coding-style`

| 规则 | 说明 |
|------|------|
| PEP 8 | 遵循 Python 编码规范 |
| 类型注解 | 所有函数签名必须有类型注解 |
| 不可变性 | 优先使用 `@dataclass(frozen=True)` |
| 格式化 | black + isort + ruff |

#### Checklist

- [ ] 函数签名有类型注解
- [ ] 单文件 < 800 行
- [ ] 函数 < 50 行
- [ ] 嵌套 < 4 层
- [ ] 无硬编码值
- [ ] 使用不可变数据结构

#### 示例

```python
# ❌ Bad
def process(data):
    result = []
    for item in data:
        if item.active:
            result.append(item)
    return result

# ✅ Good
def get_active_items(items: list[Item]) -> list[Item]:
    """Return only active items from the provided list."""
    return [item for item in items if item.is_active]
```

### 重构清理

#### 触发方式
```
"检查这段代码是否有重复/死代码"
"帮我重构这个模块"
```

#### 重构原则

1. **先有测试覆盖** - 重构前确保测试通过
2. **小步前进** - 每次只改一点
3. **持续验证** - 每步都跑测试
4. **保持行为** - 只改结构不改功能

#### 常见重构场景

| 场景 | 解决方案 |
|------|---------|
| 重复代码 | 提取函数/方法 |
| 过长函数 | 分解为小函数 |
| 过大类 | 按职责拆分 |
| 魔法数字 | 提取为常量 |
| 深层嵌套 | 提前返回/提取函数 |


### 测试类型速查

| 类型 | 标记 | 用途 | 速度 |
|------|------|------|------|
| 单元测试 | `@pytest.mark.unit` | 单个函数/类 | 快 |
| 集成测试 | `@pytest.mark.integration` | 模块间交互 | 中 |
| E2E 测试 | `@pytest.mark.e2e` | 完整用户流程 | 慢 |
| 慢测试 | `@pytest.mark.slow` | 耗时操作 | 慢 |

```bash
# 只跑单元测试
pytest -m unit

# 跳过慢测试
pytest -m "not slow"

# 只跑集成测试
pytest -m integration
```

### pytest 常用模式

```python
# 参数化测试
@pytest.mark.parametrize("input,expected", [
    ("hello", "HELLO"),
    ("world", "WORLD"),
])
def test_uppercase(input, expected):
    assert input.upper() == expected

# 异常测试
def test_divide_by_zero():
    with pytest.raises(ZeroDivisionError):
        divide(10, 0)

# Fixture 使用
@pytest.fixture
def sample_user():
    return User(name="Alice", email="alice@example.com")

def test_user_email(sample_user):
    assert "@" in sample_user.email
```

#### 安全 Checklist

##### 1. 密钥管理

- [ ] 无硬编码密钥
- [ ] 所有密钥在环境变量
- [ ] `.env` 在 `.gitignore`
- [ ] 无密钥在 git 历史

```python
# ❌ Bad
API_KEY = "sk-proj-xxxxx"

# ✅ Good
import os
API_KEY = os.environ.get("API_KEY")
if not API_KEY:
    raise ValueError("API_KEY not configured")
```

##### 2. 输入验证

- [ ] 所有用户输入经过验证
- [ ] 使用 schema 验证（Pydantic）
- [ ] 文件上传有大小/类型限制

```python
from pydantic import BaseModel, EmailStr, validator

class UserCreate(BaseModel):
    email: EmailStr
    name: str
    
    @validator('name')
    def name_not_empty(cls, v):
        if not v.strip():
            raise ValueError('name cannot be empty')
        return v
```

##### 3. SQL 注入防护

- [ ] 使用参数化查询
- [ ] 无字符串拼接 SQL

```python
# ❌ Bad
query = f"SELECT * FROM users WHERE email = '{email}'"

# ✅ Good
from sqlalchemy import text
query = text("SELECT * FROM users WHERE email = :email")
result = db.execute(query, {"email": email})
```

##### 4. 认证与授权

- [ ] Token 存在 httpOnly cookie
- [ ] 敏感操作有权限检查
- [ ] Session 管理安全

##### 5. XSS 防护

- [ ] 用户 HTML 内容经过清理
- [ ] CSP 头配置正确

##### 6. 错误信息

- [ ] 不暴露内部细节
- [ ] 详细错误只在服务端日志

```python
# ❌ Bad
return {"error": str(e), "stack": traceback.format_exc()}

# ✅ Good
logger.error(f"Internal error: {e}")
return {"error": "An error occurred. Please try again."}
```

## 五、调试与排错

### 根因分析

#### 触发方式
```
"使用 systematic-debugging 技能"
"帮我系统化调试这个问题"
```

#### 四阶段调试法

```
Phase 1: 根因调查
├── 仔细读错误信息
├── 稳定复现
├── 检查最近改动
└── 收集证据

Phase 2: 模式分析
├── 找工作示例
├── 对比差异
└── 理解依赖

Phase 3: 假设验证
├── 形成单一假设
├── 最小化测试
└── 验证后继续

Phase 4: 实施
├── 写失败测试
├── 修复根因
└── 验证修复
```

#### 铁律

```
没有根因调查，不提修复方案
```

#### 红旗信号

遇到这些想法，**回到 Phase 1**：

- "快速修复一下，以后再查"
- "试试改 X 看看行不行"
- "我不太懂但这样可能行"
- **3 次修复失败后还在继续**

### 常见调试模式

| 问题类型 | 调试策略 |
|---------|---------|
| 间歇性失败 | 添加日志，多次运行收集证据 |
| 性能问题 | profiling，找热点 |
| 内存泄漏 | 对象计数，引用追踪 |
| 并发问题 | 线程转储，竞态分析 |
| 多组件问题 | 逐层添加诊断日志 |

## 六、数据库优化

### 查询优化

#### 触发方式
```
"用 database-reviewer 审查查询"
"优化这个 SQL 查询"
```

#### 索引速查

| 查询模式 | 索引类型 | 示例 |
|---------|---------|------|
| `WHERE col = value` | B-tree | `CREATE INDEX idx ON t(col)` |
| `WHERE a = x AND b > y` | 复合 | `CREATE INDEX idx ON t(a, b)` |
| `WHERE jsonb @> '{}'` | GIN | `CREATE INDEX idx ON t USING gin(col)` |
| 时间序列范围 | BRIN | `CREATE INDEX idx ON t USING brin(col)` |

#### 复合索引原则

```sql
-- 等值列在前，范围列在后
CREATE INDEX idx ON orders (status, created_at);
-- 优化：WHERE status = 'pending' AND created_at > '2024-01-01'
```

#### 覆盖索引

```sql
-- 避免回表查询
CREATE INDEX idx ON users (email) INCLUDE (name, created_at);
-- 优化：SELECT email, name FROM users WHERE email = ?
```

#### 部分索引

```sql
-- 更小的索引，只包含需要的行
CREATE INDEX idx ON users (email) WHERE deleted_at IS NULL;
```

### N+1 问题检测

#### 触发方式
```
"检查是否有 N+1 查询问题"
```

#### SQLAlchemy 解决方案

```python
# ❌ Bad: N+1 查询
for user in session.query(User).all():
    print(user.orders)  # 每次触发一个查询

# ✅ Good: 预加载
for user in session.query(User).options(joinedload(User.orders)):
    print(user.orders)  # 一次性加载
```

### 数据类型最佳实践

| 用途 | 正确类型 | 避免 |
|------|---------|------|
| ID | `bigint` | `int`, 随机 UUID |
| 字符串 | `text` | `varchar(255)` |
| 时间戳 | `timestamptz` | `timestamp` |
| 金额 | `numeric(10,2)` | `float` |
| 布尔 | `boolean` | `varchar`, `int` |

### 性能诊断 SQL

```sql
-- 慢查询
SELECT query, mean_exec_time, calls
FROM pg_stat_statements
WHERE mean_exec_time > 100
ORDER BY mean_exec_time DESC;

-- 未索引的外键
SELECT conrelid::regclass, a.attname
FROM pg_constraint c
JOIN pg_attribute a ON a.attrelid = c.conrelid AND a.attnum = ANY(c.conkey)
WHERE c.contype = 'f'
  AND NOT EXISTS (
    SELECT 1 FROM pg_index i
    WHERE i.indrelid = c.conrelid AND a.attnum = ANY(i.indkey)
  );

-- 表膨胀
SELECT relname, n_dead_tup, last_vacuum
FROM pg_stat_user_tables
WHERE n_dead_tup > 1000
ORDER BY n_dead_tup DESC;
```

```python
from fastapi import FastAPI, HTTPException, Depends
from pydantic import BaseModel

app = FastAPI()

# 请求模型
class UserCreate(BaseModel):
    email: str
    name: str

# 响应模型
class UserResponse(BaseModel):
    id: int
    email: str
    name: str

# 依赖注入
async def get_current_user(token: str = Depends(oauth2_scheme)):
    user = verify_token(token)
    if not user:
        raise HTTPException(401, "Invalid token")
    return user

# 路由
@app.post("/users", response_model=UserResponse, status_code=201)
async def create_user(
    user: UserCreate,
    current_user: User = Depends(get_current_user)
):
    # 业务逻辑
    return await UserService.create(user)
```

### 类型注解

```python
# 基础类型
def greet(name: str) -> str:
    return f"Hello, {name}"

# 可选类型
from typing import Optional
def find_user(id: int) -> Optional[User]:
    ...

# 集合类型
def process(items: list[str]) -> dict[str, int]:
    ...

# 泛型
from typing import TypeVar, Generic
T = TypeVar('T')
class Container(Generic[T]):
    def __init__(self, value: T):
        self.value = value
```

### Dataclass

```python
from dataclasses import dataclass, field
from datetime import datetime

@dataclass(frozen=True)  # 不可变
class User:
    id: int
    name: str
    email: str
    created_at: datetime = field(default_factory=datetime.now)
    
    def __post_init__(self):
        if "@" not in self.email:
            raise ValueError(f"Invalid email: {self.email}")
```

### 上下文管理器

```python
from contextlib import contextmanager

@contextmanager
def timer(name: str):
    start = time.perf_counter()
    yield
    elapsed = time.perf_counter() - start
    print(f"{name} took {elapsed:.4f}s")

# 使用
with timer("data processing"):
    process_data()
```

### 异常处理模式

```python
# ❌ Bad: 裸 except
try:
    risky_operation()
except:
    pass

# ✅ Good: 特定异常 + 链式
try:
    data = json.loads(raw)
except json.JSONDecodeError as e:
    raise ValueError(f"Invalid JSON: {raw}") from e
```
