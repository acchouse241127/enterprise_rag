## 三、总结：后端有、前端未集成的功能

| 优先级 | 功能 | 后端接口 | 前端缺口 | 建议 |
|--------|------|----------|----------|------|
| **高** | **网址导入文档** | `POST .../documents/from-url` | 4_文档上传无「从 URL 导入」入口 | 在文档上传页增加 URL 输入框与提交 |
| **高** | **分享链接免登录查看** | `GET /conversations/share/{token}` | 无专门「分享查看页」 | 新增页面（如 `10_分享查看.py`），根据 token 调接口并展示对话 |
| **中** | 标记问题样本 | `POST /retrieval/feedback/{id}/mark-sample` | 7_看板仅展示 🔖，无切换按钮 | 在看板日志行增加「标记/取消样本」按钮 |
| **中** | 知识库更新 | `PUT /knowledge-bases/{id}` | 无编辑知识库名称等入口 | 在 3_知识库管理增加编辑表单 |
| **低** | 当前用户校验 | `GET /auth/me` | 未用 | 可选：进入需要鉴权的页面时调一次 |
| **低** | TOTP 绑定/校验 | `POST /auth/totp/setup`, `/auth/totp/verify` | 登录页有 TOTP 输入，需确认是否走这两个接口 | 确认流程并补齐 |
| **低** | 单条日志/任务详情 | `GET /retrieval/logs/{id}`, `GET /tasks/{id}` | 未用 | 可按需做详情展开 |
| **低** | 按知识库统计 | `GET /retrieval/stats/by-knowledge-base` | 未用 | 看板可按需增加筛选 |
| **低** | 问题样本列表 | `GET /retrieval/samples` | 未用 | 可做「问题样本管理」页 |

## 五、结论

- **已较好集成**：登录、知识库 CRUD、文档上传/批量/版本、问答流式、检索看板、对话管理（导出/分享/删除）、文件夹同步、异步任务、知识库编辑。
- **明显缺口**：
  1. **网址导入文档**：后端支持，前端无入口。
  2. **分享链接查看**：他人打开分享链接后，无 Streamlit 页面调用 `GET /conversations/share/{token}` 展示内容。
  3. **问题样本标记**：后端支持，前端仅展示状态，无操作按钮。
- **可选补齐**：知识库更新、`/auth/me`、TOTP 流程确认、单条详情接口等。
