# SPA 登录问题诊断与修复记录

**日期**: 2026-03-03  
**范围**: 前端 SPA 无法登录、后端启动失败、Docker 部署相关

---

## 一、前端 SPA 与其它服务的关系

### 1.1 是否需要同时启动另一个前端镜像？

**结论：不需要。**

项目中存在两个前端服务，彼此独立：

| 服务名 | 目录 | 技术栈 | 端口 | 说明 |
|--------|------|--------|------|------|
| **spa** | `frontend_spa/` | React + Vite + Nginx | 3000 | **主入口**，用户登录与主要功能 |
| **frontend** | `frontend/` | Streamlit | 8501 | 备选前端 |

- SPA 的 `depends_on` 仅包含 **backend**，不依赖 `frontend`。
- 只使用 React SPA 时，只需启动：`postgres`、`chromadb`、`redis`（若用 Celery）、**backend**、**spa**。
- 启动命令示例：`docker-compose --profile full up` 或按需单独启动上述服务。

### 1.2 登录数据流

```
用户输入 → 前端 Login.tsx → api/auth.ts → POST /api/auth/login → 后端 auth.py → 数据库验证 → 返回 JWT
```

- 前端 API 基地址：`VITE_API_BASE` 或空（开发时由 Vite 代理到 `http://localhost:8000`）。
- 登录请求路径：`/api/auth/login`，请求体 `{ username, password }`。

## 二、诊断过程（根因分析）

### 2.1 现象

- 前端 SPA 无法登录。
- Docker 中 `enterprise_rag_backend` 状态为 **unhealthy**。

### 2.2 自动执行过的诊断命令

1. **后端健康检查**：`curl http://localhost:8000/health` → 超时（后端未正常响应）。
2. **端口与容器**：8000、5432 在监听；`docker ps` 显示 backend 为 unhealthy。
3. **数据库用户**：执行 `backend` 内脚本检查 `users` 表，确认存在 **admin** 用户（active=True）。
4. **后端日志**：`docker logs enterprise_rag_backend --tail 50`，发现启动失败原因。

### 2.3 根因 1：缺少 Chunk ORM 模型

**错误信息：**

```text
sqlalchemy.exc.NoReferencedTableError: Foreign key associated with column 
'splade_embeddings.chunk_id' could not find table 'chunks' with which to 
generate a foreign key to target column 'id'
```

- `SpladeEmbedding` 模型声明了外键 `ForeignKey("chunks.id")`，但项目中没有 `chunks` 对应的 ORM 模型。
- 数据库层面已有迁移脚本 `V2.0_002_create_chunks_table.sql` 创建了 `chunks` 表（主键为 **UUID**），但 ORM 未定义，导致 SQLAlchemy 在 `Base.metadata.create_all()` 时无法解析依赖关系。

### 2.4 根因 2：Chunk 模型中保留字冲突

**错误信息：**

```text
sqlalchemy.exc.InvalidRequestError: Attribute name 'metadata' is reserved 
when using the Declarative API.
```

- 在 Chunk 模型中使用了字段名 `metadata`，与 SQLAlchemy Declarative 的保留属性冲突。
- 处理：将 ORM 字段改为 `extra_metadata`，数据库列名仍可通过 `column` 映射为 `metadata`（当前实现用 `extra_metadata` 避免保留字问题）。

### 2.5 根因 3：SpladeEmbedding.chunk_id 类型与 chunks.id 不一致

**错误信息：**

```text
sqlalchemy.exc.ProgrammingError: (psycopg2.errors.DatatypeMismatch) 
foreign key constraint "splade_embeddings_chunk_id_fkey" cannot be implemented
DETAIL:  Key columns "chunk_id" and "id" are of incompatible types: integer and uuid.
```

- 迁移脚本中 `chunks.id` 为 **UUID** 类型。
- `SpladeEmbedding` 中 `chunk_id` 定义为 **Integer**，导致外键类型不匹配。
- **待修复**：将 `SpladeEmbedding.chunk_id` 改为 **UUID** 类型（与 `chunks.id` 一致）。

### 2.6 其它发现

- **tiktoken 未安装**：后端在导入 `app.rag.chunker` 时依赖 `tiktoken`，容器内未安装会导致 `ModuleNotFoundError: No module named 'tiktoken'`。已在容器内执行 `pip install tiktoken` 作为临时修复；建议在 `backend` 的 `requirements.txt` 或 Dockerfile 中正式加入该依赖。
- **默认管理员账号**：脚本 `backend/scripts/create_admin_user.py` 创建的用户名为 **admin**，密码为 **password123**。需先运行该脚本（或等价逻辑）才能登录。

## 三、已完成的代码修改

### 3.1 新增 Chunk ORM 模型

**文件**: `backend/app/models/chunk.py`

- 定义表 `chunks`，与迁移 `V2.0_002_create_chunks_table.sql` 一致。
- 主键 `id` 为 **UUID**。
- 注意：ORM 中避免使用保留字 `metadata`，使用字段名 **extra_metadata** 映射 JSONB 列（若需与现有 DB 列名 `metadata` 一致，可再通过 `mapped_column(..., name="metadata")` 指定）。

### 3.2 注册 Chunk 并导出

**文件**: `backend/app/models/__init__.py`

- 增加：`from .chunk import Chunk`，并在 `__all__` 中加入 `"Chunk"`。
- 保证 `SpladeEmbedding` 在解析外键时能找到 `chunks` 表对应的模型。

### 3.3 SpladeEmbedding 类型与重复导入

**文件**: `backend/app/models/splade_embedding.py`

- 删除重复的 `from app.core.database import Base`。
- 将 `chunk_id` 由 **Integer** 改为 **UUID**（`PgUUID(as_uuid=True)`），与 `chunks.id` 一致，消除 DatatypeMismatch。

### 3.4 小结

- Chunk 模型已添加并注册，解决了 “找不到 chunks 表” 的 NoReferencedTableError。
- Chunk 中保留字冲突已通过 `extra_metadata` 解决。
- SpladeEmbedding.chunk_id 已改为 UUID，与迁移脚本一致。重启 backend 并确认健康后即可验证 SPA 登录。

## 五、验证登录的步骤

1. **启动服务**（Docker）：  
   `docker-compose --profile full up -d` 或按需启动 postgres、chromadb、redis、backend、spa。

2. **等待 backend 健康**：  
   `docker ps` 中 `enterprise_rag_backend` 为 **healthy**。

3. **确认管理员用户**：  
   执行 `create_admin_user`（见 4.2）。

4. **测试登录接口**：  
   ```bash
   curl -X POST http://localhost:8000/api/auth/login \
     -H "Content-Type: application/json" \
     -d "{\"username\":\"admin\",\"password\":\"password123\"}"
   ```  
   应返回 `code: 0` 及 `data.access_token`。

5. **前端 SPA 登录**：  
   打开 SPA（如 http://localhost:3000），使用 **admin** / **password123** 登录。
