# V2.0 API Documentation

## Verification API

### POST /retrieval/feedback

Submit user feedback for a retrieval result.

**Request Body:**
```json
{
  "retrieval_log_id": 1,
  "feedback_type": "helpful",
  "rating": 1,
  "reason": "Answer was accurate",
  "comment": "Optional comment"
}
```

## QA Response Fields (V2.0)

The QA response now includes quality metrics:

- confidence_score: float (0.0-1.0)
- confidence_level: string (high/medium/low)
- faithfulness_score: float (0.0-1.0)
- verification_action: string (pass/filter/retry/refuse)
- refusal_message: string (optional, when refused)
