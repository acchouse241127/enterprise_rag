## Task B1-1：PostgreSQL 自定义镜像（pg_jieba）

**文件**：
- create: `docker/postgres/Dockerfile`
- create: `docker/postgres/init-extensions.sql`
- modify: `docker-compose.yml`（postgres 使用 build 而非 image）

**步骤**：

1. 创建 `docker/postgres/Dockerfile`（基于 postgres:16，从源码编译 pg_jieba）
2. 创建 `docker/postgres/init-extensions.sql`（`CREATE EXTENSION IF NOT EXISTS pg_jieba;`）
3. 修改 `docker-compose.yml`：postgres 服务改为 `build: ./docker/postgres`，image 改为 `enterprise_rag_postgres:pg_jieba`
4. 运行 `docker compose build postgres` 验证构建成功
5. 运行 `docker compose up -d postgres` 验证启动
6. 进入容器执行 `psql -U enterprise_rag -d enterprise_rag -c "CREATE EXTENSION IF NOT EXISTS pg_jieba;"` 验证扩展可用

**验收**：`docker compose build postgres` 成功；`docker compose up -d postgres` 成功；`pg_jieba` 扩展可创建
