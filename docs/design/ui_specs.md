## 2. 问答页 (QA Chat Page)

**页面路径**: `/chat` (或作为默认首页)

### 2.1 页面布局

```
+-------------------------------------------------------+
|  Sidebar (侧边栏)      |  Main Content (主内容区)      |
|                        |                               |
|  [系统 Logo & 名称]    |  [当前知识库名称] (H2)        |
|  [当前用户]            |                               |
|  --------------------  |  +-------------------------+  |
|  [知识库选择下拉框]    |  |  (聊天记录区域)         |  |
|                        |  |  User: ...              |  |
|  [新建对话按钮]        |  |  Assistant: ...         |  |
|  --------------------  |  |  [引用来源折叠面板]     |  |
|  [历史对话列表]        |  |                         |  |
|  - 对话 A              |  +-------------------------+  |
|  - 对话 B              |                               |
|  ...                   |  +-------------------------+  |
|                        |  |  (输入区域)             |  |
|                        |  |  [输入框] [发送按钮]    |  |
|                        |  +-------------------------+  |
+-------------------------------------------------------+
```

### 2.2 组件详细说明

1.  **侧边栏 (`st.sidebar`)**:
    *   **Logo & 名称**: 使用 `st.image` 或 `st.title` 展示。
    *   **知识库选择**: `st.selectbox("选择知识库", options=kb_list)`。
    *   **新建对话**: `st.button("➕ 新建对话", type="primary")`，点击清空当前会话状态。
    *   **历史对话**: `st.radio` 或自定义组件展示历史会话列表，点击切换 `session_state.current_chat_id`。

2.  **主内容区**:
    *   **标题**: `st.header(f"💬 与 {current_kb_name} 对话")`。
    *   **聊天记录**: 使用 `st.chat_message("user")` 和 `st.chat_message("assistant")` 循环渲染历史消息。
    *   **引用来源**: 在 Assistant 回复下方，使用 `st.expander("📚 引用来源")` 包裹引用列表。每条引用显示 `文档名 (页码/位置) - 相似度`。

3.  **输入区域**:
    *   **输入框**: `st.chat_input("请输入您的问题...")`。
    *   **发送逻辑**: 用户回车或点击发送后，显示 `st.spinner("正在思考...")`，调用后端流式接口，逐字显示回复。

### 2.3 交互逻辑

*   **切换知识库**: 切换后自动清空当前对话记录，或提示用户是否开启新对话。
*   **流式输出**: 必须支持打字机效果，提升用户体验。
*   **错误处理**: 若后端返回错误，在聊天区域显示 `st.error("请求失败: ...")`，允许用户重试。

## 3. 知识库管理页 (Knowledge Base Management)

**页面路径**: `/knowledge_base`

### 3.1 页面布局

```
+-------------------------------------------------------+
|  Sidebar (侧边栏)      |  Main Content (主内容区)      |
|                        |                               |
|  (同上)                |  [页面标题: 知识库管理] (H1)  |
|                        |                               |
|                        |  +-------------------------+  |
|                        |  |  [新建知识库表单]       |  |
|                        |  |  名称 *                 |  |
|                        |  |  描述                   |  |
|                        |  |  [创建按钮]             |  |
|                        |  +-------------------------+  |
|                        |                               |
|                        |  [知识库列表标题] (H2)        |
|                        |  [搜索框]                     |
|                        |                               |
|                        |  +-------------------------+  |
|                        |  |  [知识库卡片 1]         |  |
|                        |  |  名称: 合同库           |  |
|                        |  |  文档数: 12             |  |
|                        |  |  [查看文档] [删除]      |  |
|                        |  +-------------------------+  |
|                        |  |  [知识库卡片 2] ...     |  |
|                        |  +-------------------------+  |
+-------------------------------------------------------+
```

### 3.2 组件详细说明

1.  **新建区域**:
    *   使用 `st.form("create_kb_form")`。
    *   **名称**: `st.text_input("知识库名称 *")`。
    *   **描述**: `st.text_area("描述")`。
    *   **提交**: `st.form_submit_button("创建", type="primary")`。

2.  **列表区域**:
    *   **搜索**: `st.text_input("🔍 搜索知识库", placeholder="输入名称...")`。
    *   **展示**: 建议使用 `st.columns(3)` 循环展示卡片，或使用 `st.dataframe` (若字段较多)。
    *   **卡片内容**:
        *   **名称**: `st.subheader`。
        *   **信息**: `st.caption(f"文档数: {doc_count} | 创建时间: {create_time}")`。
        *   **操作**:
            *   `st.button("📂 管理文档", key=f"btn_manage_{kb_id}")` -> 跳转至文档管理页。
            *   `st.button("🗑️ 删除", key=f"btn_delete_{kb_id}")` -> 弹出确认或直接删除（需二次确认逻辑）。

### 3.3 交互逻辑

*   **创建成功**: 显示 `st.success` 并自动刷新列表。
*   **删除保护**: 删除前需检查知识库内是否有文档，若有则提示先删除文档或强制删除（需谨慎）。

## 4. 文档上传页 (Document Upload)

**页面路径**: `/documents` (或 `/upload`)

### 4.1 页面布局

```
+-------------------------------------------------------+
|  Sidebar (侧边栏)      |  Main Content (主内容区)      |
|                        |                               |
|  (同上)                |  [页面标题: 文档管理] (H1)    |
|                        |  [当前知识库: 合同库] (H3)    |
|                        |                               |
|                        |  +-------------------------+  |
|                        |  |  (上传区域)             |  |
|                        |  |  [文件拖拽框]           |  |
|                        |  |  支持 PDF, DOCX, ...    |  |
|                        |  |  [上传按钮]             |  |
|                        |  +-------------------------+  |
|                        |                               |
|                        |  [文档列表标题] (H2)          |
|                        |  [搜索/筛选栏]                |
|                        |                               |
|                        |  +-------------------------+  |
|                        |  |  [文档数据表格]         |  |
|                        |  |  ID | 文件名 | 状态 ... |  |
|                        |  +-------------------------+  |
|                        |  [分页组件]                   |
+-------------------------------------------------------+
```

### 4.2 组件详细说明

1.  **上传区域**:
    *   **文件选择**: `st.file_uploader("上传文档", type=["pdf", "docx", "txt", "md"], accept_multiple_files=True)`。
    *   **上传按钮**: `st.button("开始上传", type="primary")`。点击后显示进度条 `st.progress`。

2.  **列表区域**:
    *   **筛选**: `col1, col2 = st.columns(2)` -> `st.text_input("搜索文件名")`, `st.selectbox("状态筛选", ["全部", "处理中", "成功", "失败"])`。
    *   **表格**: `st.dataframe(docs_df, use_container_width=True, column_config={...})`。
        *   **状态列**: 使用 `st.column_config.StatusColumn` 显示图标（✅, 🔄, ❌）。
        *   **操作列**: Streamlit 表格内操作较弱，建议在表格下方提供“选中操作”或使用 `st.data_editor` 的 checkbox 列来批量删除。

### 4.3 交互逻辑

*   **异步处理**: 上传后后端异步解析，前端需轮询状态或提供“刷新”按钮 `st.button("🔄 刷新列表")`。
*   **大文件限制**: 前端提示文件大小限制（如 50MB）。
*   **解析失败**: 状态显示为“失败”时，鼠标悬停或点击详情应显示错误原因。

---

## 5. 总结

请开发团队 (C2) 依据以上规范进行页面开发。如有技术实现上的困难（如 Streamlit 组件限制），请及时反馈给 R4 进行设计调整。

**核心原则**:
1.  **一致性**: 保持所有页面的布局、颜色、字体一致。
2.  **反馈**: 任何操作都要有明确的反馈（成功/失败/加载）。
3.  **简洁**: 避免过多复杂的装饰，专注于功能和内容。
