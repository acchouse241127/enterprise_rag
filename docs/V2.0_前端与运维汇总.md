## 二、镜像说明

### 2.1 enterprise_rag-worker:latest

- **与项目关系**：属于 V2.0 项目，对应 `docker-compose.yml` 中的 `worker` 服务。
- **用途**：Celery 异步任务（文档解析、向量化等）。
- **启动方式**：因配置了 `profiles: [full]`，需使用：
  ```bash
  docker-compose --profile full up -d worker
  ```
- **状态**：已按需启动。

### 2.2 postgres:16-alpine

- **与项目关系**：作为**构建依赖**，用于构建自定义镜像 `enterprise_rag_postgres:pg_jieba`。
- **说明**：项目实际运行时使用的是 `enterprise_rag_postgres:pg_jieba`，不是直接运行 `postgres:16-alpine`。该基础镜像可保留用于未来重建，或删除以节省空间。

## 三、双前端策略与 Streamlit 淘汰

### 3.1 两个前端对比

| 对比项 | Streamlit (frontend/) | React SPA (frontend_spa/) |
|--------|------------------------|---------------------------|
| 技术栈 | Python + Streamlit | React + TypeScript + Tailwind |
| 镜像大小 | **773MB** | **93.3MB**（约 8 倍差距） |
| UI 风格 | 简洁功能性 | 现代化、企业级 |
| 检索质量看板 | 功能完整（含趋势、日志、按 KB 统计、问题样本） | 原为简化版，已规划迁移完整功能 |
| 运维/扩展 | 受框架限制 | 完全可控、易扩展 |

### 3.2 镜像体积差异原因

| 因素 | Streamlit | React SPA |
|------|-----------|-----------|
| 基础镜像 | python:3.11-slim（约 150MB） | nginx:alpine（约 25MB） |
| 运行时 | 完整 Python + Streamlit 及数据科学依赖（pandas、numpy 等） | 仅 nginx + 静态文件 |
| 构建方式 | 单阶段，镜像内含全部依赖 | 多阶段构建，最终镜像只含静态资源 |

**结论**：React SPA 生产镜像更小、更易维护，适合作为唯一前端。

### 3.3 是否保留 Streamlit 的结论

从**未来迭代开发与运维**角度，建议**不保留 Streamlit 前端**：

| 维度 | 建议 |
|------|------|
| 运维成本 | 去掉 773MB 镜像与额外容器，降低资源占用 |
| 维护成本 | 单一前端代码库，避免双倍功能与 UI 维护 |
| 用户体验与扩展性 | 以 React SPA 为主，完全自主可控 |

**执行顺序**（已约定）：
1. **先做**：将 Streamlit 检索质量看板完整功能迁移到 React SPA，并完成测试验证。
2. **后做**：停止并移除 Streamlit 容器/镜像及相关配置。

## 四、检索质量看板迁移（React SPA）

### 4.1 迁移范围（来自 Streamlit `7_检索质量看板.py`）

- **筛选条件**：知识库下拉、时间范围（7/14/30 天）、刷新。
- **核心指标卡片**：总查询数、平均最高分、平均响应时间、无用反馈率、有用/无用反馈数、问题样本数、平均返回块数。
- **查询趋势**：按日统计的查询量柱状图、每日平均分数折线图（可用纯 CSS/简单条形或后续接入轻量图表库）。
- **检索日志列表**：分页、按反馈状态筛选（全部/有反馈/无反馈/有用/无用）、单条日志的👍/👎/🔖 操作（添加反馈、标记问题样本）。
- **按知识库统计**：表格展示各知识库的查询数、平均分、平均耗时、有用/无用反馈数。
- **问题样本列表**：仅展示已标记为问题样本的检索记录，支持刷新。

### 4.2 后端 API（已存在，无需改后端）

- `GET /api/retrieval/stats`：汇总统计（支持 knowledge_base_id、start_date、end_date）。
- `GET /api/retrieval/stats/by-date`：按日统计（支持 knowledge_base_id、days）。
- `GET /api/retrieval/stats/by-knowledge-base`：按知识库统计。
- `GET /api/retrieval/logs`：检索日志列表（分页、has_feedback、feedback_type）。
- `GET /api/retrieval/logs/{log_id}`：单条日志详情。
- `POST /api/retrieval/feedback`：添加反馈（helpful/not_helpful，需带 rating 等）。
- `POST /api/retrieval/feedback/{feedback_id}/mark-sample`：标记/取消问题样本。
- `GET /api/retrieval/samples`：问题样本列表。

### 4.3 前端已完成的准备工作

- **类型**（`frontend_spa/src/api/types.ts`）：`RetrievalStats`、`DailyStats`、`KbStats`、`RetrievalLog`、`RetrievalLogFeedback`、`ProblemSample` 及对应 Response 类型。
- **API 封装**（`frontend_spa/src/api/dashboard.ts`）：`getRetrievalStats`、`getStatsByDate`、`getStatsByKnowledgeBase`、`getRetrievalLogs`、`getRetrievalLogDetail`、`addFeedback`、`markFeedbackAsSample`、`getProblemSamples`、`getDashboardStats`（兼容原有调用）。

### 4.4 待完成的前端工作

- 重构 `Dashboard.tsx`：接入上述 API，实现筛选器与核心指标卡片。
- 查询趋势：使用简单图表或轻量图表库展示每日查询量与平均分。
- 检索日志列表：表格/列表 + 分页 + 反馈状态筛选 + 单条操作的👍/👎/🔖。
- 按知识库统计：表格展示。
- 问题样本列表：列表展示 + 刷新。
- 无数据/未开启检索日志时的提示（与 Streamlit 文案一致：完成 RAG 问答后刷新、检查 RETRIEVAL_LOG_ENABLED 等）。
