## 问题列表

### 问题 #1 | 引用来源需支持一键访问源文件

- **发现日期**：2026-02-20
- **验收项**：2_问答对话 - 引用来源展示
- **严重程度**：高
- **状态**：已修复

**现象**：引用来源仅显示「文档 ID: X | 片段: Y」纯文本，无法点击跳转到源文件对应位置。

**复现步骤**：
1. 在「问答对话」页选择知识库并提问
2. 查看回答下方的「引用来源」区域
3. 引用项为灰色纯文本，无法点击

**预期**：引用来源应可点击，能直接打开/定位到源文件对应片段。

**实际**：仅显示 ID 与片段号，无交互。

**相关代码/文件**：
- `frontend/pages/2_问答对话.py`：render_citations、聊天历史中的引用 expander

**解决方案**：为每条引用增加「👁️ 预览原文」「📥 下载」按钮，通过已有 `/documents/{id}/preview` 和 `/documents/{id}/download` API 实现一键访问源文件。

**实施**：
- `frontend/pages/2_问答对话.py`：新增 `_render_citation_with_actions()`，在引用来源 expander 中为每条引用添加预览/下载
- `backend/app/api/qa.py`：新增 `_enrich_citations_with_filename()`，在 ask 与 stream 响应中为 citations 补充 filename

### 问题 #2 | 质量看板提示语导致用户误以为问答页有刷新按钮

- **发现日期**：2026-02-20
- **验收项**：2.3 问题样本标记（7_检索质量看板）
- **严重程度**：中
- **状态**：已修复

**现象**：提示语「完成 RAG 问答后，请点击上方「🔄 刷新」查看统计」易被理解为在问答页点击刷新，但问答页无刷新按钮，用户完成问答后无法找到入口进行检索质量分析。

**复现步骤**：
1. 在「问答对话」页完成若干次 RAG 问答
2. 进入「检索质量看板」页
3. 若指标全为 0，看到提示「请点击上方「🔄 刷新」」
4. 用户误以为需回到问答页点击刷新

**预期**：明确告知刷新按钮位于当前（质量看板）页，且可从问答页便捷跳转至看板。

**实际**：提示语中「上方」指本页筛选区，但未明确说明「本页」或「检索质量看板」，易产生歧义。

**相关代码/文件**：
- `frontend/pages/7_检索质量看板.py`：约 269 行提示语

**解决方案**：修改提示为「请点击本页筛选区右侧的「🔄 刷新」按钮」；并在问答页增加「查看检索质量统计」入口，引导用户跳转到看板页。

**实施**：
- `frontend/pages/7_检索质量看板.py`：将「上方」改为「本页筛选区右侧」
- `frontend/pages/2_问答对话.py`：侧边栏增加 `st.page_link` 指向检索质量看板
