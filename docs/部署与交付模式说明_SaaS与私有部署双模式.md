# 部署与交付模式说明：SaaS 与私有部署双模式

本文档整合产品交付策略、租户模式下文件夹同步方案、三种模式的用户场景、稳定性与效率分析，以及双模式（SaaS + 私有部署 ToB）的技术与产品要点。便于产品、售前与研发统一理解与落地。

---

## 一、三种交付模式总览

| 模式 | 数据位置 | 目标用户 | 典型定价 |
|------|----------|----------|----------|
| **SaaS** | 云端 | 个人 / 小团队 | 订阅制（如 ¥99/月） |
| **私有部署** | 客户服务器 | ToB 企业 | 年费 / 买断（如 ¥50,000/年） |
| **完全本地** | 用户电脑 | 高隐私 / 离线场景 | 买断（如 ¥999） |

### 1.1 产品策略（双模式重点）

- **SaaS**：用于个人或小团队，开箱即用，多设备同步与协作。
- **私有部署 + 配置云端 LLM API**：用于 ToB 企业，数据在客户侧，检索在本地，问答可走云端大模型，兼顾性能与合规。

## 二、租户运营模式下的文件夹同步

### 2.1 问题背景

当前文件夹同步基于**服务端路径**：后端直接读取 `directory_path`。适用于私有部署（用户控制服务器），**不适用于 SaaS**（用户无法访问服务器文件系统）。

### 2.2 方案对比

| 方案 | 做法 | 优点 | 缺点 | 适用 |
|------|------|------|------|------|
| **A** | 浏览器选择文件夹，递归上传 | 无需安装，纯 Web | 大文件量时性能与体验一般 | SaaS 轻量同步 |
| **B** | 桌面同步客户端监控本地目录，通过 API 同步 | 自动、增量、体验好 | 需开发与分发客户端 | 企业级 SaaS |
| **C** | 对接 OneDrive / 阿里云 OSS 等 | 用户熟悉，无需装客户端 | 需各平台 OAuth/集成 | 已用云存储的用户 |
| **D** | WebDAV / SFTP 凭证，后端拉取 | 通用 | 凭证与安全需谨慎 | 技术用户 |

### 2.3 数据存储位置说明

| 数据类型 | 方案 A/B（SaaS 同步） | 私有部署（服务端路径） | 完全本地 |
|----------|------------------------|------------------------|----------|
| 原始文件 | 本地保留 + 服务器有副本 | 仅在客户服务器 | 100% 本地 |
| 向量/索引 | 服务器 | 客户服务器 | 本地 |
| 对话记录 | 服务器 | 客户服务器 | 本地 |

**结论**：方案 A/B 为混合存储——原始文件用户本地仍有，但上传后服务器会存副本；若需「数据完全在用户电脑」，需采用完全本地化部署（见下文完全本地模式）。

## 三、三种模式的用户使用场景

### 3.1 方案一：SaaS（个人 / 小团队）

**用户画像**：个人知识工作者、小团队，无技术背景，希望多设备访问。

**典型场景**：产品经理小王整理产品文档

- 打开浏览器访问 `app.your-rag.com`，登录后创建知识库「产品需求文档」。
- 使用「同步本地文件夹」选择 `D:\产品文档\2026年需求`，系统发现 156 个文件，点击「开始同步」，等待上传与解析完成。
- 知识库就绪后提问：「我们 Q2 的核心功能是什么？」系统返回答案与引用。
- 在咖啡厅用手机继续访问同一知识库；新增文档可直接在页面上传。
- 将知识库分享给同事，实现协作查阅。

**特点**：无需安装、多设备同步、需联网；文档会存储在云端，适合接受云存储的用户。

---

### 3.2 方案二：私有部署（ToB 企业）

**用户画像**：企业 IT、有合规要求，数据需留在公司内部。

**典型场景**：某金融公司部署内部知识库

- **IT 管理员**：在公司服务器执行 `docker-compose up -d`，访问 `http://192.168.1.100:8000`，创建管理员账号，可选对接 LDAP。
- **配置同步**：公司文档在 `/data/company-docs/`，在管理后台创建知识库「内部制度」，配置同步目录（如 `/data/sync/1`），按界面生成的 Docker 挂载命令在宿主机执行并重启容器，点击「立即同步」。
- **员工使用**：员工通过内网访问，域账号登录，在「内部制度」知识库中提问（如请假流程），系统返回答案并引用《员工手册》。
- **持续运行**：定时自动同步新文档，IT 查看检索质量看板；所有数据仅在公司服务器，可配置云端 LLM API 做问答。

**特点**：数据在公司内网、可对接内部认证、可完全离线或仅 LLM 走公网；需要服务器与基础运维能力。

---

### 3.3 方案三：完全本地（高隐私 / 离线）

**用户画像**：对隐私极度敏感或需离线使用（如律师、医生、研究人员）。

**典型场景**：律师老李的本地知识库

- 安装「Enterprise RAG 桌面版」，首次启动自动下载本地向量模型与可选本地 LLM（如 Qwen2.5-7B）。
- 添加知识库并选择本地文件夹 `E:\案件资料\2026年`，解析与向量化均在本地完成。
- 提问与回答全部在本地完成，无数据上传；出差无网络时仍可正常使用。
- 卸载后，原始文件仍保留在用户指定目录。

**特点**：零上传、可完全离线；依赖本机算力，多设备与团队协作需另做设计。

---

### 3.4 三种模式对比小结

| 维度 | SaaS | 私有部署 | 完全本地 |
|------|------|----------|----------|
| 安装难度 | 无需安装 | 中等（Docker） | 简单（安装包） |
| 硬件要求 | 无 | 服务器 | 中高配置电脑 |
| 网络要求 | 必须联网 | 内网即可 | 可完全离线 |
| 数据位置 | 云端 | 客户服务器 | 用户电脑 |
| 协作能力 | 强 | 中 | 无 |
| 维护成本 | 无 | 中 | 低 |
| 适合规模 | 个人/小团队 | 企业 | 个人 |

## 四、稳定性与效率分析

### 4.1 稳定性

| 维度 | SaaS | 私有部署 | 完全本地 |
|------|------|----------|----------|
| 服务可用性 | 高（多节点、专业运维） | 中（单点依赖运维） | 中（依赖单机） |
| 网络依赖 | 高（断网不可用） | 低（内网即可） | 无 |
| 硬件故障影响 | 低 | 中 | 高 |
| 数据安全 | 依赖服务商 | 客户自主控制 | 最高（本地） |

**结论**：从服务可用性与运维成熟度看，**SaaS 最稳定**；从数据自主权看，私有部署与完全本地更可控。

### 4.2 效率

| 维度 | SaaS | 私有部署 | 完全本地 |
|------|------|----------|----------|
| 检索速度 | 高 | 最高（内网 + 可选 GPU） | 中（受本机性能限制） |
| 问答生成 | 高（云端大模型） | 高（可配云端 LLM） | 中（本地模型） |
| 文件上传/同步 | 中（受带宽影响） | 高（内网/本地路径） | 最高（本地读写） |
| 并发能力 | 高 | 中 | 低 |

**结论**：**私有部署 + 云端 LLM API** 在检索延迟与问答质量上可达到最佳平衡；SaaS 次之；完全本地更侧重隐私与离线，性能受本机限制。

### 4.3 综合建议

- **追求稳定与省心**：选 **SaaS**。
- **追求性能与数据可控**：选 **私有部署 + 配置云端 LLM API**。
- **追求极致隐私与离线**：选 **完全本地**。

## 五、双模式实施要点（SaaS + 私有部署 ToB）

### 5.1 产品矩阵

```
                    企业级需求
                        ↑
    ┌───────────────────┼───────────────────┐
    │                   │                   │
    │   私有部署         │    SaaS           │
    │   (ToB 企业版)     │   (个人/小团队)    │
    │   数据在客户侧     │   数据在云端      │
    │   + 云端 LLM API   │                   │
    └───────────────────┼───────────────────┘
                        │
                    ┌───┴───┐
                    │完全本地│（可选，高隐私场景）
                    └───────┘
```

### 5.2 技术架构共性

- **同一套代码**：后端、前端、Worker 共用同一仓库；通过环境变量与配置区分 SaaS / 私有部署。
- **私有部署**：现有 Docker Compose 已支持；配置 `LLM_PROVIDER`、`LLM_API_KEY`、`LLM_BASE_URL` 等即可使用云端 LLM；文件夹同步使用服务端路径 + Docker 挂载（见下节）。
- **SaaS**：多租户隔离（如 `tenant_id`）、订阅与限流、文件夹同步采用「浏览器批量上传」或后续同步客户端/云存储集成。

### 5.3 私有部署：文件夹同步与 Docker 友好化

- 同步目录为**容器内路径**，例如 `/data/sync/{知识库ID}`。
- 前端提供「生成容器路径」与 **Docker 挂载命令**（`docker run -v` 与 `docker-compose` 片段）及一键复制，用户按说明在宿主机执行挂载并重启后端/Worker。
- 详见：`docs/V2.0_前端功能迭代说明_检索看板与文件夹同步.md` 第三节。

### 5.4 配置差异摘要

| 项目 | SaaS | 私有部署 ToB |
|------|------|----------------|
| 多租户 | 启用 `tenant_id` 隔离 | 可选（单租户可不用） |
| 认证 | 统一登录 / 可选 SSO | 本地账号 / 可选 LDAP |
| 文件夹同步 | 浏览器上传或同步客户端 | 服务端路径 + 宿主机挂载 |
| LLM | 云端 API（统一配置） | 云端 API（客户自配 Key） |
| 数据与备份 | 运营方负责 | 客户自行备份与合规 |
