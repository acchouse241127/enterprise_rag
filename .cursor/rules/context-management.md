# 上下文管理规则 / Context Management Rules

> **约束**：Cursor 每个 agent 对话框上下文约 200K tokens，长会话易导致上下文爆炸、响应变慢或丢失关键信息。
> **目标**：通过会话边界、检查点、精简手牌，使开发过程不依赖单会话长上下文。

---

## 一、核心原则

| 原则 | 说明 |
|------|------|
| **会话即边界** | 每会话视为独立 200K 预算，不追求单会话完成全部任务 |
| **文件即记忆** | 关键状态写入文件，新会话通过读取文件恢复，而非依赖对话历史 |
| **子代理即隔离** | 每任务派遣新子代理，子代理不继承父会话的累积上下文 |
| **精简即安全** | 手牌只传任务必要信息，不传完整 PRD/TRD/设计文档 |

---

## 二、分批会话 + 检查点

每 **3-5 任务** 切换会话，避免单会话累积超过 200K。

**流程**：执行一批任务 → 写入检查点文件 → 关闭会话 → 新会话读取检查点继续

---

## 三、检查点文件规范

**目录**：`docs/plans/checkpoints/`（或项目约定的计划目录）

**命名**：`{项目或阶段}-{批次}.md`，如 `V2.0-B1-infra-done.md`

**模板**：

```markdown
# 检查点：{阶段/批次名称}

**完成时间**：YYYY-MM-DD
**完成任务**：任务编号列表
**Git 基准**：`git rev-parse HEAD` 输出

## 已变更关键文件
- 路径列表

## 验证命令与结果
\`\`\`
命令及输出
\`\`\`

## 下一阶段前置条件
- 条件列表

## 下一批任务
任务编号及简要说明
```

检查点文件纳入版本管理，便于新会话恢复上下文。

---

## 四、子代理手牌精简

派遣 implementer 子代理时，**不要**传入完整 PRD/TRD/设计文档。**只传**：

| 内容 | 说明 |
|------|------|
| 任务全文 | 从计划文件摘录的该任务完整步骤 |
| 相关文件路径 | 需创建/修改的文件 |
| 验收标准 | 该任务的通过条件 |
| 必要片段 | 仅该任务相关的 1-2 段规格（如 SQL 模板、接口定义） |

子代理若需更多上下文，由其自行 codebase_search，遵循 iterative-retrieval：先宽搜，再评估，再精炼。

---

## 五、战略压缩时机

在以下节点建议手动 `/compact` 或关闭会话重开：

| 时机 | 动作 |
|------|------|
| 规划完成后、执行前 | 压缩规划讨论，保留计划文件引用 |
| 每 3-5 任务完成 | 写入检查点 → 关闭会话 → 新会话从检查点继续 |
| 每批次 code review 后 | 压缩 review 讨论，保留「已修复/待修复」结论 |
| 调试完成后 | 压缩调试过程，保留根因与修复方案 |

参考技能：`strategic-compact`

---

## 六、计划文档拆分

大型实现计划按阶段拆成多个文件，每会话只加载当前阶段：

| 文件类型 | 内容 | 用途 |
|----------|------|------|
| overview | 总览、批次、依赖图、检查点索引 | 每会话必读 |
| 阶段文件 | 当前阶段任务的详细步骤 | 仅加载当前阶段 |
| 检查点 | 最新完成批次的摘要 | 新会话恢复用 |

单会话加载：overview + 1 个阶段文件 + 最新检查点，控制单次读取量。

---

## 七、新会话启动话术

新会话开始时，使用类似话术快速恢复上下文：

```
继续 {项目/阶段} 开发。请读取：
1. docs/plans/{overview}.md
2. docs/plans/checkpoints/{最新检查点}.md
3. docs/plans/{当前阶段}.md

从任务 {任务编号} 开始，使用 executing-plans 技能执行本阶段任务。
```

---

## 八、与执行策略的对应

| 执行方式 | 上下文策略 |
|----------|------------|
| **子代理驱动（本会话）** | 父会话只保留 TodoWrite + 当前任务索引；每任务新子代理零继承；每 5 任务写检查点并建议 /compact |
| **并行会话（executing-plans）** | 每批次 = 新会话；批次间通过检查点文件交接；天然避免上下文累积 |

**推荐**：在 200K 限制下，优先采用「并行会话 + 分批执行」，每 3 任务一批，批次间通过检查点文件交接。

---

## 九、相关技能

- `strategic-compact`：在逻辑边界建议手动压缩
- `iterative-retrieval`：子代理按需检索上下文，避免一次性加载全部
- `executing-plans`：分批执行，每批新会话
- `subagent-driven-development`：每任务新子代理，上下文隔离
