# Fix Acceptance Issue / 修复验收问题

处理手工验收发现的问题，按交互式流程完成记录与修复。

## 工作流程

### 第一步：收集信息
用户提供：
- **截图**：出现问题处的界面（必选）
- **错误代码**：控制台/终端的报错内容（如有）
- **自然语言描述**：问题现象、复现步骤、预期行为

### 第二步：澄清确认
通过 1～2 轮问答确认：
- 问题发生的页面/功能点
- 操作步骤是否完整
- 预期与实际结果的差异
- 是否可稳定复现

### 第三步：记录入库
将澄清后的问题写入：
- 文件：`docs/issues/acceptance-issues.md`
- 格式：按下方模板追加一条记录
- 同时更新：`docs/reports/phase-4-manual-acceptance.md` 中的问题汇总表

### 第四步：生成方案
- 定位相关代码（前端/后端/配置）
- 分析根因
- 提出具体修改建议（含文件、行号、修改内容）
- 征得用户同意后实施修复

---

## 问题记录模板

```markdown
### 问题 #N | [简短标题]
- **发现日期**：YYYY-MM-DD
- **验收项**：来自 manual-acceptance-v2.md 的章节编号
- **严重程度**：高/中/低
- **状态**：待修复 | 已修复 | 已关闭

**现象**：（澄清后的描述）

**复现步骤**：
1. ...
2. ...

**预期**：...

**实际**：...

**相关代码/文件**：（待定位后补充）

**解决方案**：（待分析后补充）
```

---

## 使用方式

1. 在对话中发送 `/fix-acceptance-issue` 或直接说明「我要提交一个验收问题」
2. 粘贴截图、错误代码（如有），并用自然语言描述问题
3. 按 AI 的澄清问题作答
4. 确认记录无误后，等待解决方案并决定是否执行修复
